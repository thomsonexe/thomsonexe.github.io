file_magic

A file was uploaded to the server with the wrong extension. The analyst flagged it as suspicious but couldn't identify it. Use magic bytes and Linux file tools to find out what it really is and extract its contents.

Objectives

  • Identify the real file type of suspicious.doc
  • Rename or handle it appropriately to access its contents
  • Find the hidden flag inside

Get the Challenge Files

$ cd ~/challenges/file-magic/
$ cat README.txt

Tools You'll Need

  • file suspicious.doc
  • xxd suspicious.doc | head
  • unzip
Open Terminal

strings_attached

A firmware image was recovered from a compromised IoT device. The development team believes a secret was accidentally compiled into the binary during a debug build. Find it and decode it.

Objectives

  • Extract all readable strings from firmware.bin
  • Identify the suspicious encoded string
  • Decode it to recover the flag

Get the Challenge Files

$ cd ~/challenges/strings-attached/
$ cat README.txt

Tools You'll Need

  • strings firmware.bin
  • strings firmware.bin | grep =
  • echo "..." | base64 -d

init.flag

A user account has been configured on a Debian system. Explore the filesystem to find what was left behind. Some flags are in plain sight — others require decoding.

Objectives

  • Find the hidden file in the user's home directory
  • Decode the base64-encoded string inside it
  • Locate a second flag buried deeper in the filesystem
  • Submit both flags in the format flag{...}

Get the Challenge Files

$ wget https://thomson.cx/challenges/init-flag.tar.gz
$ tar -xzf init-flag.tar.gz && cd init-flag/

Tools You'll Need

  • ls -la — list hidden files
  • find / -name "*.flag" 2>/dev/null
  • echo "..." | base64 -d
Open Terminal

log_trace

A web server was breached overnight. You've been handed the Apache access logs from the incident window. Find the attacker, trace their steps, and determine what they exfiltrated.

Objectives

  • Identify the attacker's IP address
  • Determine which vulnerability was exploited
  • Find the first file accessed post-exploitation
  • Identify the exfiltration method used

Get the Challenge Files

$ wget https://thomson.cx/challenges/log-trace.tar.gz
$ tar -xzf log-trace.tar.gz && cd log-trace/

Tools You'll Need

  • grep, awk, cut
  • sort | uniq -c | sort -rn
  • sed for log field parsing

Custom Medium 100 pts

hash_hunt

A credential database was dumped during a breach. The passwords are hashed — but not all algorithms are equal. Identify the type and crack the target entry to recover the plaintext.

Objectives

  • Identify the hash algorithm used in the database
  • Crack the target hash to recover the plaintext password
  • Find which other account reuses the same password

Get the Challenge Files

$ wget https://thomson.cx/challenges/hash-hunt.tar.gz
$ tar -xzf hash-hunt.tar.gz && cd hash-hunt/

Tools You'll Need

  • hashid or hash-identifier
  • john --wordlist=/usr/share/wordlists/rockyou.txt
  • grep for cross-referencing entries
Open Terminal
